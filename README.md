# Oracle Bones to LLMs

**Oracle Bones to LLMs** is a small interactive web project that compares two systems for producing interpretable answers under uncertainty.

A user asks a question about judgment, conflict, risk, or the future. The site then returns two responses side by side:

- a **Shang-inspired oracle-bone divination response**
- an **LLM-style advisory response**

The user then decides which response feels:

- more trustworthy
- more meaningful
- more intelligent
- more useful

## Core idea

This project is not arguing that oracle bones were “early AI.”  
Instead, it compares **two cultural technologies of interpretation**.

Oracle bones were part of a highly regularized Shang ritual system involving turtle plastrons and ox scapulae with inscriptions used for divination. Large language models, by contrast, generate fluent answers by statistically reorganizing patterns from massive corpora of human language.

The comparison asks:

> When an output feels wise or intelligent, where does that feeling come from?  
> The machine itself, or the cultural form of the answer?

## Why I made it

This project responds to course readings on Alan Turing’s imitation game, the Dartmouth AI research program, and recent arguments that large AI models are best understood as **cultural and social technologies** rather than just intelligent agents.

Turing’s framework makes intelligence a matter of successful textual performance. Oracle-bone divination suggests a different standard: credibility grounded in ritual, inscription, and interpretive authority.

## Built with

- HTML
- CSS
- JavaScript

No external APIs were used.

## How to run

1. Save the file as `index.html`
2. Open it in a browser
3. Enter a question and compare the two systems

## What surprised me

The oracle-style answers can feel deep because they are compressed and ambiguous.  
The LLM-style answers can feel intelligent because they are smooth, explanatory, and balanced.

That difference is the point.
